# 魔方网表 RBAC 权限查看器 - 实现文档

## 项目概述

基于魔方网表单页应用，实现指定账号在当前空间的权限查询。支持查询：
- 直接针对用户的权限
- 用户所在的用户组权限
- 用户所在的组织结构节点权限
- 系统保留组权限（空间管理员、代管理员、审计员、流程管理员）
- 系统保留角色权限（未登录用户、已登录用户、创建人）

## 技术栈

- React + TypeScript + Vite
- shadcn/ui 组件库
- Tailwind CSS

## 核心模块说明

### 1. 鉴权模块 (auth.ts)

**功能**：JWT Token 管理，支持可配置启用/禁用

**配置方式**：
- URL参数：`?auth=false` 禁用鉴权
- URL参数：`?username=x&password=y` 自定义账号
- 代码默认：启用状态，测试账号 `mfiv202602130001 / 123456`

**核心函数**：
```typescript
authenticatedFetch(url, options)  // 自动带Token的fetch
getAuthHeaders()                   // 获取鉴权头
isTokenAuthEnabled()                // 检查是否启用
getTokenStatus()                    // 获取Token状态
manualLogin(username, password)     // 手动登录
getAuthConfig()                     // 获取当前配置
```

**Token机制**：
- 30分钟有效期，代码按28分钟提前过期
- 自动刷新机制：检查 `j_refresh` 参数
- 存储：localStorage

### 2. 权限API封装 (permission.ts)

**API路径规则**：
- `/magicflu/service/s/{spaceId}/...` → XML响应（权限接口）
- `/magicflu/service/s/json/{spaceId}/...` → JSON响应（名称查询）

**核心函数**：

| 函数 | 用途 |
|------|------|
| `fetchUserAuthorities` | 查询用户直接权限主体 |
| `fetchOrgAuthorities` | 查询用户组织结构权限主体 |
| `fetchLoggedInUserAuthority` | 已登录用户权限(typeId=5) |
| `fetchAuthorityByTypeId` | 按类型查询权限主体 |
| `fetchSystemGroupAuthorities` | 系统保留组(4,7,9,10) |
| `fetchItemActionGroups` | 操作模板（位->名称映射） |
| `fetchItemTypeAcls` | 对象类权限 |
| `fetchItemAcls` | 对象权限 |
| `fetchItemName` | 获取对象名称（按类型分发） |

**XML解析**：
- 使用 `DOMParser` 解析XML响应
- 路径：`feed entry content authority ...`

### 3. 业务逻辑 Hook (usePermissionQuery.ts)

**查询流程**：

```
1. 收集权限主体
   ├─ fetchUserAuthorities (直接用户)
   ├─ fetchOrgAuthorities (组织结构)
   ├─ fetchLoggedInUserAuthority (已登录用户)
   ├─ fetchAuthorityByTypeId (未登录用户, typeId=3)
   ├─ fetchAuthorityByTypeId (创建人, typeId=6)
   └─ fetchSystemGroupAuthorities (系统组)

2. 查询对象类权限
   └─ fetchItemTypeAcls (每个authId)

3. 查询操作组模板
   └─ fetchItemActionGroups (按itemTypeId)

4. 查询对象权限
   └─ fetchItemAcls (每个authId)

5. 查询对象名称
   └─ fetchItemName (按itemTypeId)
```

**返回数据结构**：
```typescript
interface PermissionSummary {
  digitalId: string
  spaceId: string
  authorities: Authority[]           // 权限主体列表
  itemTypeAcls: Map<number, ItemTypeAcl[]>   // 对象类权限
  itemAcls: Map<string, ItemAclWithName[]>  // 对象权限
  actionGroups: Map<number, ActionGroup[]>   // 操作组模板
  itemPermissionGroups: ItemPermissionGroup[] // 扁平化的对象权限
  stats: {
    authorityCount: number           // 权限主体数
    itemPermissionCount: number      // 对象权限数
    itemTypePermissionCount: number  // 对象类权限数
  }
  error?: string
}
```

### 4. 页面组件 (App.tsx)

**布局**：
- 顶部导航栏（固定）
- 搜索栏（spaceId + digitalId 输入）
- 统计卡片（3个指标）
- 主体区域：左侧表格 + 右侧信息栏
- 三个Tab：对象权限 / 对象类权限 / 权限主体

**功能**：
- 输入spaceId和用户登录账号查询
- 对象权限筛选（对象类型、权限主体类型、关键字搜索）
- 权限位解析显示（ACL bitstring → 权限名称）
- 右侧显示当前用户会话信息和组织树

## Vite配置

```typescript
// vite.config.ts
proxy: {
  '/magicflu/service': {
    target: 'http://demo.mf999.com',
    changeOrigin: true,
    secure: false,
  },
  '/magicflu/jwt': {
    target: 'http://demo.mf999.com',
    changeOrigin: true,
    secure: false,
  }
}
```

**关键点**：
- Target指向 `http://demo.mf999.com`，不是 `/magicflu`
- 透传 `Authorization` 头
- 生产模式 base 为 `/magicflu/`

## 部署说明

根据文档要求：
1. 服务请求地址使用相对路径 `/magicflu`（已配置）
2. 生产部署去掉Token鉴权（使用URL参数 `?auth=false`）
